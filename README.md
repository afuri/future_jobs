# Каталог профессий будущего  

сайт-референс: https://atlas100.ru/catalog/
документ-референс: future_jobs.pdf, skills_of_future.pdf
База данных профессий: jobs.json

Web‑приложение на Flask для навигации по профессиям будущего

## Технологии
- Python 3.12
- Flask
- Jinja2
- HTML/CSS (минималистичный интерфейс)
- База данных jobs.json


## Как должен выглядеть “качественный, но управляемый” функционал

### 1) Каталог будущих профессий

* карточки профессий (название, кратко “зачем миру”, сфера, тип деятельности, ключевые навыки)
* детальная страница профессии:
  **skills**, **инструменты**, **типичные задачи**, **маршрут входа**, **риски/этика**, “почему может появиться”

### 2) Фильтры (простые, но мощные)

Фильтрация по:

* **сфере** (bio/ai/edu/cyber/creative/urban/space/medicine/energy…)
* **типу деятельности** (анализ, коммуникация, разработка, дизайн, управление, исследование)
* **ключевому навыку** (Python, data, storytelling, UX, системное мышление и т.д.)
* **уровню математики/коммуникаций** (низк/сред/высок)

Это выглядит просто, но даёт нормальную логику.

### 3) Умный поиск

* поиск по словам в названии/описании/навыках
* нормализация запроса (lower, ё→е, удаление мусора, разбор на токены)
* “похожие запросы” (минимально — по синонимам из маленького словарика)

### 4) Мини-опрос (5–7 вопросов) → рекомендации

Это главный “вау”: пользователь отвечает на вопросы → получает топ-10 профессий **с объяснением**.

Опрос не должен быть психологическим тестом — лучше “про предпочтения деятельности”:

* “Мне нравится: объяснять / исследовать / строить системы / создавать визуал / договариваться”
* “Мне комфортнее: определённость / неопределённость”
* “Хочу больше: людей / данных / технологий / творчества”

### 5) Объяснимость 

В выдаче не просто список, а:

* **почему** профессия рекомендована (совпали сфера + деятельность + навыки)
* какие навыки “не хватает” (skill gap)

## Модульная архитектура (чтобы ученики правили только бизнес-логику)

Рекомендую так (Flask/Jinja — отдельно, логика — отдельно):

```
app.py
templates/...
data/
  jobs.json         # база будущих профессий (можно sqlite позже)
logic/
  normalize.py      # нормализация запросов
  search.py         # поиск
  filters.py        # фильтрация
  quiz.py           # обработка ответов опроса
  recommend.py      # скоринг + топ
  explain.py        # объяснение “почему”
  gaps.py           # skill gap + план действий
  taxonomy.py       # словари сфер/навыков/синонимов
```

> Для конкурсного занятия это идеально: каждая пара берёт один модуль и делает “кусок настоящего продукта”.

## “Не слишком сложно”, но “качественно”: минимальный MVP


1. **Каталог + фильтры**
2. **Поиск**
3. **Опрос → топ-10 + объяснение**
4. **Skill gap + 3 шага “что делать”**


---

## Обязательно

* пользователь отвечает на 6 вопросов → получает **персональный топ-10 будущих профессий**
* у каждой профессии есть **объяснение** и **шаги**, что попробовать в школе
* фильтр по “ключевому навыку: Python” даёт другой список, чем “коммуникация”
* поиск по “экология” находит профессии даже если слово только в `why_future`
* гибрид: **70% сфера/деятельность + 30% навыки**, это золотая середина для школы и конкурса.
